DocStream {{ .Chart.AppVersion }} installed as "{{ .Release.Name }}".

Migrations and Kafka topics ran as pre-install hooks, so the pipeline is ready
once the pods report Ready:

  kubectl get pods -l app.kubernetes.io/instance={{ .Release.Name }} -w

Reach the APIs by port-forwarding (no ingress by default):

  kubectl port-forward svc/{{ include "docstream.fullname" . }}-gateway {{ .Values.services.gateway.port }}:{{ .Values.services.gateway.port }}
  kubectl port-forward svc/{{ include "docstream.fullname" . }}-query {{ .Values.services.query.port }}:{{ .Values.services.query.port }}

Then, in another shell:

  # write path
  curl -F "file=@/tmp/lease.txt" http://localhost:{{ .Values.services.gateway.port }}/documents
  curl http://localhost:{{ .Values.services.gateway.port }}/jobs/<job_id>

  # read path
  curl http://localhost:{{ .Values.services.query.port }}/documents
  curl "http://localhost:{{ .Values.services.query.port }}/search?q=security+deposit"
  curl -X POST http://localhost:{{ .Values.services.query.port }}/ask \
    -H 'Content-Type: application/json' \
    -d '{"question":"What is the security deposit?"}'
